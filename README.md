# 播客转逐字稿 (podcast-transcribe)

将播客音频转为可读的逐字稿，包含完整工作流。

## 设计目的

这个音频转逐字稿的目的是：
1. **保存精彩内容**：把播客中很精彩、很有思想内容、很有世界观方法论的内容固定下来。因为播客时间太长（1-2小时），精彩观点可能只集中在某几分钟或几十分钟
2. **延续讨论**：节目中嘉宾没有讨论到或阐述不清楚的地方，可以把逐字稿喂给 AI，让用户可以继续讨论或深入探讨

## 功能特性

| 步骤 | 功能 |
|------|------|
| 1 | 下载音频（支持小宇宙等播客平台） |
| 2 | 音频自动切分（超过1小时切40分钟/段） |
| 3 | Whisper Large-V3 转写（中文效果最好） |
| 4 | 断点续传（检测已完成的 part >1KB，跳过） |
| 5 | 合并所有段落 |
| 6 | 搜索嘉宾及节目信息（用于区分发言人） |
| 7 | LLM 智能洗稿（去语气词、整理逻辑、区分发言人） |
| 8 | 超长文本分段处理（10万字以上也能处理） |

## 额外功能

### 按时间段提取核心内容
用户可以指定一个时间范围（如 10:30 - 15:00），从逐字稿中提取该时间段的内容，并让 AI 提炼核心观点。

### 继续深度讨论
拿到逐字稿后，可以：
- 让 AI 提炼某个时间段的核心观点
- 让 AI 扮演节目里的某个人物继续讨论
- 针对"浅尝辄止"的点让 AI 继续深挖

## 使用方式

### 用户输入格式
```
用户：帮我转写这个播客 https://xxx.com/episode/xxx 对话人是李继刚和孟岩
```

### 完整工作流
1. 用户提供播客网页 URL + 对话人信息（选填）
2. 解析音频下载地址并下载
3. 音频切分（>1小时切40分钟/段）
4. Whisper Large-V3 逐段转写
5. 断点续传（检测已完成的 part）
6. 合并所有段落
7. 搜索嘉宾及节目信息（区分发言人）
8. LLM 清洗（结合搜索信息，自动标注发言人）
9. 输出到 podcasts/ 目录

### 发言人区分机制

Whisper 无法按音色区分发言人，通过以下方式解决：

1. **搜索节目信息**：自动搜索小宇宙节目页、单集详情、搜索引擎，获取嘉宾介绍、章节信息
2. **智能推断**：清洗时让 LLM 结合节目信息与逐字稿内容自行推断发言人
3. **格式输出**：
   - 【确定嘉宾】：可直接判断的用此格式
   - 【推测：XXX】：无法确定时用此格式标注

### 清洗 Prompt（结合搜索信息的增强版）

清洗时同时输入：
- 原始逐字稿
- 搜索到的节目信息（嘉宾介绍、章节等）

模型输出示例：
```
【李继刚】：我们今天来聊聊...
【孟岩】：好啊，我也想听听...
【主持人】：欢迎各位来到...
【推测：嘉宾A】：根据上下文推断...
```

**第一步：分段总结（每 10000 字一段）**

如果用户提供对话人信息，按以下格式整理：

```
请阅读以下逐字稿片段，总结其中的核心观点和关键信息。

对话人信息：[例如：李继刚（也叫继刚）、孟岩（也叫老孟）]

要求：
- 用简洁的语言概括这段内容的要点
- 保留关键数据和核心观点
- 不要口水话
- 修正识别错误的人名（根据对话人信息）

逐字稿内容：
[粘贴这段的逐字稿]
```

**第二步：合并总结并整体清洗**

把所有分段总结合并后，再进行最终清洗：

```
你是一个专业的文字编辑，擅长整理对话录音。

请整理以下多段总结，遵循以下规则：

1. **删除语气助词**：嗯、啊、这个、那个、然后、其实、就是、基本上、大概等
2. **去除口水话**：重复的表述、口齿不清的碎片
3. **完善修正表达**：只保留最终正确的表达
4. **整理逻辑顺序**：重新排列使其符合逻辑
5. **保留核心**：人名、专业术语、关键观点、具体数据必须保留

请直接输出整理后的内容，不要添加任何评论。

分段总结内容：
[粘贴所有分段总结]
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 下载失败 | 让用户提供其他 URL |
| 音频切分失败 | 提示用户检查文件 |
| Whisper 被 kill | 提示"进程被系统终止"，自动断点续传 |
| 合并失败 | 提示用户检查各 part 文件 |
| 清洗失败 | 明确提示用户，可查看原始稿 |

## 依赖

- `ffmpeg`：音频切分（`brew install ffmpeg`）
- `openai-whisper`：本地转写模型
  ```bash
  pip install openai-whisper
  # 首次运行自动下载 large-v3 模型（约3GB）
  ```
- **注意**：Whisper 在 CPU 上运行，>1小时音频必须切分否则会被系统 kill

## 输出格式

- 原始逐字稿：`{EP}_{标题}_逐字稿.txt`
- 清洗后逐字稿：`{EP}_{标题}_逐字稿_清洗.txt`
